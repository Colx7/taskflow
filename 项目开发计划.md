## 1. 项目概述

### 1.1 一句话介绍

TaskFlow 是一个基于 FastAPI + Vue 3 全栈开发的 AI 智能任务管理系统，集成通义千问大模型实现任务自动分类、智能优先级推荐和周报自动生成，是展示 AI 全栈工程能力的实习作品集项目。

### 1.2 核心亮点

| 亮点 | 说明 |
|------|------|
| **AI 智能分类** | 用户输入任务标题和描述后，LLM 自动识别类别、设置优先级 |
| **AI 周报生成** | 一键生成结构化周报，包含完成情况、逾期提醒、改进建议 |
| **全栈工程能力** | FastAPI + Vue 3 + MySQL + Docker + JWT + 前后端分离架构 |
| **RESTful API 设计** | 规范的 RESTful 接口，Swagger 文档自动生成 |
| **响应式 UI** | Element Plus 组件库，适配桌面端主流分辨率 |

## 2. 数据库设计

### 2.1 ER 关系描述

```
users (1) ──────< tasks >────── (M) categories
  │                       │
  │                       │
  │                      (M)
  │                        │
  └──────────────────── task_tags >────── (1) tags
```

- **users** 与 **tasks**: 一对多（一个用户可以创建多个任务）
- **categories** 与 **tasks**: 一对多（一个分类下可以有多条任务）
- **tags** 与 **tasks**: 多对多（通过中间表 task_tags 关联）

### 2.2 表结构定义

#### 表 1: users（用户表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 加密后的密码 |
| avatar_url | VARCHAR(500) | NULL | 头像 URL（预留） |
| created_at | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

索引: `idx_username`, `idx_email`

#### 表 2: categories（分类表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) | UNIQUE, NOT NULL | 分类名称 |
| icon | VARCHAR(20) | NULL | 图标标识（预留） |
| color | VARCHAR(20) | NULL | 颜色标识（预留） |
| sort_order | INT | NOT NULL, DEFAULT 0 | 排序顺序 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |

预置数据: 工作(1)、学习(2)、生活(3)、健身(4)、其他(5)

#### 表 3: tags（标签表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(30) | UNIQUE, NOT NULL | 标签名称 |
| color | VARCHAR(20) | NULL | 标签颜色 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |

#### 表 4: tasks（任务表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| title | VARCHAR(200) | NOT NULL | 任务标题 |
| description | TEXT | NULL | 任务详细描述 |
| status | ENUM('pending','in_progress','completed','cancelled') | NOT NULL, DEFAULT 'pending' | 任务状态 |
| priority | ENUM('low','medium','high','urgent') | NOT NULL, DEFAULT 'medium' | 优先级 |
| category_id | INT | FK -> categories.id, NOT NULL | 所属分类 |
| user_id | BIGINT | FK -> users.id, NOT NULL | 所属用户 |
| due_date | DATE | NULL | 截止日期 |
| tags | JSON | NULL | 关联的标签名称数组（如 ["urgent","deadline"]） |
| ai_suggestion | JSON | NULL | AI 分析结果缓存（category, priority, reason） |
| created_at | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

索引: `idx_user_status`, `idx_user_category`, `idx_due_date`, `idx_priority`

#### 表 5: task_tags（任务-标签关联表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| task_id | BIGINT | FK -> tasks.id, PK | 任务 ID |
| tag_id | INT | FK -> tags.id, PK | 标签 ID |

联合主键: `(task_id, tag_id)`

### 2.3 SQL 建表语句（参考）

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(500) DEFAULT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL UNIQUE,
    icon VARCHAR(20) DEFAULT NULL,
    color VARCHAR(20) DEFAULT NULL,
    sort_order INT NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE tags (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(30) NOT NULL UNIQUE,
    color VARCHAR(20) DEFAULT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    description TEXT DEFAULT NULL,
    status ENUM('pending','in_progress','completed','cancelled') NOT NULL DEFAULT 'pending',
    priority ENUM('low','medium','high','urgent') NOT NULL DEFAULT 'medium',
    category_id INT NOT NULL,
    user_id BIGINT NOT NULL,
    due_date DATE DEFAULT NULL,
    tags JSON DEFAULT NULL,
    ai_suggestion JSON DEFAULT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_status (user_id, status),
    INDEX idx_user_category (user_id, category_id),
    INDEX idx_due_date (due_date),
    INDEX idx_priority (priority)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE task_tags (
    task_id BIGINT NOT NULL,
    tag_id INT NOT NULL,
    PRIMARY KEY (task_id, tag_id),
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

请根据提供的任务统计数据，生成一份结构化的周报。

def build_task_summary(tasks: list[dict]) -> str:
    """将任务列表转为文本摘要供 LLM 消费"""
    completed = [t for t in tasks if t['status'] == 'completed']
    pending = [t for t in tasks if t['status'] == 'pending']
    in_progress = [t for t in tasks if t['status'] == 'in_progress']
    overdue = [t for t in tasks if t['status'] != 'completed' and t.get('due_date') and dateutil.parse(t['due_date']) < datetime.now()]

    summary_lines = [
        f"本周总任务: {len(tasks)}",
        f"已完成: {len(completed)}",
        f"进行中: {len(in_progress)}",
        f"待办: {len(pending)}",
        f"逾期: {len(overdue)}",
        "",
        "已完成任务列表:",
    ]
    for t in completed[-10:]:  # 最近10条
        summary_lines.append(f"- [{t['category']}] {t['title']} (完成于 {t['updated_at']})")

    summary_lines.append("")
    summary_lines.append("逾期任务:")
    for t in overdue:
        summary_lines.append(f"- [{t['category']}] {t['title']} (截止日 {t['due_date']})")

    return "\n".join(summary_lines)

### 5.4 错误处理与降级策略

```python
async def _call_llm(self, prompt: str) -> str:
    """带重试和降级的 LLM 调用"""
    max_retries = 2
    for attempt in range(max_retries + 1):
        try:
            response = await self.client.post(
                "/compatible-mode/v1/chat/completions",
                headers={
                    "Authorization": f"Bearer {self.api_key}",
                    "Content-Type": "application/json"
                },
                json={
                    "model": self.model,
                    "messages": [
                        {"role": "system", "content": self.system_prompt},
                        {"role": "user", "content": prompt}
                    ],
                    "temperature": float(os.getenv("AI_TEMPERATURE", 0.3)),
                    "max_tokens": int(os.getenv("AI_MAX_TOKENS", 2048)),
                    "response_format": {"type": "json_object"}
                }
            )
            response.raise_for_status()
            result = response.json()
            return result["choices"][0]["message"]["content"]

        except httpx.HTTPStatusError as e:
            if e.response.status_code == 429:  # 限流
                await asyncio.sleep(2 ** attempt)
                continue
            raise
        except httpx.TimeoutException:
            if attempt < max_retries:
                await asyncio.sleep(1)
                continue
            raise
        except Exception:
            raise

    # 降级: 返回本地规则分类
    return self._fallback_classify(title, description)

def _fallback_classify(self, title: str, description: str) -> dict:
    """API 不可用时的降级策略: 基于关键词的规则匹配"""
    keywords = {
        "工作": ["会议", "汇报", "文档", "代码", "项目", "bug", "上线", "评审"],
        "学习": ["阅读", "课程", "考试", "证书", "教程", "笔记", "复习"],
        "生活": ["购物", "买菜", "打扫", "缴费", "预约", "聚餐"],
        "健身": ["跑步", "健身", "锻炼", "游泳", "瑜伽", "举铁"],
    }
    for category, kws in keywords.items():
        for kw in kws:
            if kw in title or kw in (description or ""):
                return {
                    "category": category,
                    "priority": "medium",
                    "reason": f"[降级] 基于关键词 '{kw}' 匹配到 {category}",
                    "suggested_tags": []
                }
    return {
        "category": "其他",
        "priority": "medium",
        "reason": "[降级] 未匹配到关键词，使用默认分类",
        "suggested_tags": []
    }
```

## 附录 B: 常用命令速查

### 后端

```bash
# 激活虚拟环境
uv venv && source .venv/bin/activate   # macOS/Linux
uv venv && .venv\Scripts\activate      # Windows

# 安装依赖
uv pip install -r requirements.txt

# 启动开发服务器
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 数据库迁移
alembic revision --autogenerate -m "description"
alembic upgrade head

# 运行测试
pytest tests/ -v --cov=app --cov-report=term-missing
```

### 前端

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 生产构建
npm run build

# 代码检查
npm run lint
```

### Docker

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f backend

# 停止所有服务
docker-compose down

# 清理数据卷
docker-compose down -v
```
